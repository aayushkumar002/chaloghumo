# Contributing to ChaloGhumo

Welcome! We are excited to have you contribute to ChaloGhumo. To maintain high standards of code quality and architectural integrity, we follow a strict contribution workflow.

## 🚫 Direct Pushes are Forbidden

**Never push directly to the `main` branch.** All changes must be submitted via Pull Requests (PRs) to ensure proper peer review and automated testing.

## 🌿 Branching Protocols

We follow a structured branching convention. All new branches must follow the format:

`<type>/<owner-name>/<brief-description>`

### Supported Types:

- `feature/`: For new functionality or major enhancements.
- `bug/`: For resolving unexpected behavior or defects.
- `fix/`: For minor patches or quick repairs.
- `docs/`: For documentation updates.
- `refactor/`: For code improvements that do not change behavior.

### Example:

- `feature/aligned-automation/context-engine`
- `fix/john-doe/weather-api-timeout`

## 📬 Pull Request Process

1. **Create a branch** locally following the protocols above.
2. **Commit your changes** with clear, descriptive messages.
3. **Push to remote** and open a Pull Request.
4. **Link relevant documentation** or issues in the PR description.
5. **Await Review**: At least one owner must approve the PR before it can be merged.

## 🤝 Community Standards

By contributing to this project, you agree to uphold the innovative and collaborative spirit of the **Aligned Automation** team.
